## Guideliens

- Make the smallest possible change to achieve the goal (e.g. if a script only needs to be converted to PowerShell, do that without refactoring the logic or structure).

- Don't make any unncessary changes to the existing code- stick solely to the tasks you were explicitly assigned to do.

- Dont make any assumptions- proceed with only the information you have been given. If you need more information, ask for it.

- Always validate and support assertions with references to specific lines of code, documentation, or other sources. Don't make claims without backing them up with speifici firth-hand evidence.

- Never commit changes that you havent personally validated. Always run, build, lint, test, etc. the code yourself, according to the proscribed validation rules to ensure it works as expected before committing.
